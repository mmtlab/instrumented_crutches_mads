MADS Configuration File

The main configuration file is /usr/local/etc/mads.ini. This file is critical for sensor calibration and system behavior.

File Location

# Edit configuration
sudo nano /usr/local/etc/mads.ini

Configuration Sections

[broker]

Hust ZeroMQ broker settings:

[broker]
# Listen address for all agents
uri = "tcp://*:9092"
# Number of message worker threads
workers = 4

[coordinator]

Acquisition session management:

[coordinator]
# Default data directory
data_dir = "/home/pi/data"
# Session index file
index_file = "index.json"
# Auto-generate session IDs
auto_session_id = true

[load_cell_calibration]

Critical for accurate force measurements:

[load_cell_calibration]
# Tip load cell calibration factors (N per ADC unit)
# These values are obtained from sensor calibration
tip_left_scale = 0.01247
tip_right_scale = 0.01251
tip_left_offset = 32768
tip_right_offset = 32765

# Handle load cell calibration
handle_left_scale = 0.01248
handle_right_scale = 0.01252
handle_left_offset = 32768
handle_right_offset = 32768
Important: Obtain calibration factors from load cell datasheets or factory calibration certificates. Incorrect values will result in incorrect force measurements.

[sensor_rates]

Data acquisition sampling rates (Hz):

[sensor_rates]
# Load cell sampling rate
loadcell_rate = 1000  # Hz

# PPG (pulse oximetry) sampling rate
ppg_rate = 100       # Hz (options: 50, 100, 167, 200, 400, 600, 800, 1000)

# Eye tracker sampling rate
eye_tracker_rate = 200  # Hz (Pupil Neon native: 200 Hz)

[network]

Network and connectivity settings:

[network]
# Master crutch IP (for slave to connect)
master_ip = "10.42.0.1"
# Slave crutch IP
slave_ip = "10.42.0.2"
# Broker connection URL
broker_url = "tcp://10.42.0.1:9092"

# Network timeout (ms)
connection_timeout = 5000
reconnect_interval = 2000

Sensor Calibration

Load Cell Calibration Procedure

  1. Prepare: Known weights (e.g., 10 N, 20 N, 30 N standards)
  2. Zero: Record ADC values with no load applied
  3. Load: Apply known force and record ADC value
  4. Calculate: Scale = Force / (ADC - ADC_zero)
  5. Verify: Test multiple points linearity
  6. Update: Enter values in mads.ini

Example Calibration

# If ADC range is 0-65535 (16-bit)
# And 0g = 32500, 100kg = 32000
# Scale = (100 * 9.81) / (32500 - 32000)
#       = 981 / 500
#       = 1.962 N/unit

tip_left_scale = 1.962
tip_left_offset = 32500

PPG Sensor Configuration

MAX30100 sensor parameters in code or config:

Parameter Values Default Effect
led_current_red 4.4-50.0 mA 50.0 Red LED brightness (higher = more signal but more noise)
led_current_ir 4.4-50.0 mA 50.0 Infrared LED brightness
pulse_width 200, 400, 800, 1600 µs 1600 Integration time (higher = lower noise but higher latency)

Sampling Rates & Bandwidth

Recommended Configuration

Sensor Recommended Rate Data Rate (KB/s) Notes
Tip Load Cell (x2) 1000 Hz ~32 High frequency for detailed force transients
Handle Load Cell (x2) 1000 Hz ~32 Same as tip for consistency
PPG (x2) 100 Hz ~6.4 Sufficient for HR detection; lower reduces noise
Eye Tracker 200 Hz ~25 Native Pupil Neon rate; fixed by hardware
Total — ~95 ~570 MB per 100 seconds of recording

Network Configuration

Static IP Setup (Recommended)

Configure static IPs on both Raspberry Pi boards for stable operation:

# Edit netplan configuration
sudo nano /etc/netplan/01-netcfg.yaml

Example configuration:

network:
  version: 2
  ethernets:
    eth0:
      dhcp4: no
      addresses:
        - 10.42.0.1/24
      routes:
        - to: default
          via: 10.42.0.254
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]

Apply changes:

sudo netplan apply

WiFi Configuration (Optional)

# For eye tracker connection to Pupil Neon WiFi
sudo nano /etc/netplan/02-wifi.yaml
network:
  version: 2
  wifis:
    wlan0:
      dhcp4: yes
      access-points:
        "CrutchWiFi":
          password: "secure_password"

Systemd Service Configuration

Service File Template

Location: /etc/systemd/system/mads-{agent}.service

[Unit]
Description=MADS {Agent Name}
After=network.target mads-broker.service
Wants=mads-broker.service

[Service]
Type=simple
ExecStart=/usr/local/bin/{agent_executable} --config /usr/local/etc/mads.ini
Restart=on-failure
RestartSec=5
StandardOutput=journal
StandardError=journal
SyslogIdentifier={agent}

[Install]
WantedBy=multi-user.target

Common Service Commands

# Check service status
sudo systemctl status mads-coordinator.service

# View service logs
sudo journalctl -u mads-coordinator.service -f

# Restart service
sudo systemctl restart mads-coordinator.service

# Restart all MADS services
sudo systemctl restart 'mads-*.service'
