Project Description

The Instrumented Crutches system is a comprehensive multi-sensor platform designed to collect synchronized biomechanical and behavioral data from users during gait analysis studies. The system captures:

  • Force distribution on both crutches (tip and handle load cells)
  • Real-time eye gaze and head movement
  • Heart rate and pulse oximetry
  • Synchronized inertial measurement data

All data is collected in real-time, synchronized across devices, and stored in HDF5 format for post-acquisition analysis.

Software Agents

Coordinator

Central command and control. Manages acquisition sessions, coordinates experiments, and orchestrates data recording across all agents.

Tip & Handle Loadcell

Reads force measurements from load cells via ADC, performs filtering, and publishes calibrated force values.

PPG Agent

Interfaces with MAX30100 sensor for pulse oximetry. Starts/stops monitoring on demand.

Eye Tracker

Connects to Pupil Neon glasses, streams gaze coordinates and timestamps to local buffer.

HDF5 Writer

Subscribes to all sensor topics and writes synchronized data to hierarchical HDF5 files.

Status Handler

Monitors agent health and system errors. Publishes status updates for diagnostics.

Web Server

FastAPI-based interface for real-time monitoring, calibration, and data download.

UPS HAT

Power management and battery monitoring for continuous operation.

Data Flow

The system follows a publish/subscribe messaging pattern:

  1. Acquisition: Sensors (load cells, PPG, eye tracker) publish raw/processed data at configurable rates
  2. Processing: Filter agents may process or aggregate data before publication
  3. Storage: HDF5 Writer subscribes to all topics and writes timestamped data to files
  4. Visualization: Web Server and clients subscribe to data streams for real-time display
  5. Download: Users can download recorded sessions in HDF5 or CSV format
Synchronization: All timestamps are synchronized via the MADS broker to ensure sub-millisecond alignment across distributed sensors.

Research Context

This system was designed for observational gait analysis studies investigating:

  • Force distribution patterns during different walking/movement conditions
  • Correlation between visual attention and balance control
  • Individual variability in load-sharing strategies
  • Real-world mobility challenges in dual-task scenarios

The modular, agent-based architecture allows easy addition of new sensors or processing stages without modifying core infrastructure.

Master/Slave Architecture

The system uses a master/slave topology for robust, scalable operation:

  • Master (Right Crutch): Hosts central services (web server, coordinator, data storage, eye-tracking control)
  • Slave (Left Crutch): Collects local sensor data and publishes to master via network link

This design ensures:

  • Single point of user interaction (master web interface)
  • Centralized data storage with efficient indexing
  • Reduced bandwidth and processing on resource-constrained slave
  • Easy failover: slave can operate independently for local data buffering
