Installation Guide

Prerequisites: Both Raspberry Pi Zero 2 boards must have Raspberry Pi OS Bookworm or later, with SSH access and network connectivity configured.

Prerequisites

  • Two Raspberry Pi Zero 2 W boards with microSD cards (64GB+)
  • Raspberry Pi OS Bookworm or later
  • Network connectivity (Ethernet or WiFi)
  • SSH access enabled on both boards
  • Basic Linux command-line skills
# Check Raspberry Pi OS version
cat /etc/os-release

# Update system packages
sudo apt update && sudo apt upgrade -y

Install MADS Framework

Install MADS v2.0.0 on both boards. Refer to MADS Documentation for latest instructions.

# Download and install MADS 2.0.0
cd /tmp
wget https://github.com/pbosetti/MADS/releases/download/v2.0.0/mads_2.0.0_armv7l.deb
sudo dpkg -i mads_2.0.0_armv7l.deb

# Verify installation
mads -v

Clone Repository

Clone this repository on both boards (use the same location on both for consistency):

cd ~
git clone https://github.com/mmtlab/instrumented_crutches_mads.git
cd instrumented_crutches_mads

Install Agents

Each agent must be installed separately following its README. Install only applicable agents for each crutch role:

Master Crutch (Right) - Install All Agents

cd coordinator && make install && cd ..
cd handle_loadcell && make install && cd ..
cd tip_loadcell && make install && cd ..
cd hdf5_writer && make install && cd ..
cd status_handler && make install && cd ..
cd web_server && pip install -r requirements.txt && cd ..
cd eye_tracker && pip install -r requirements.txt && cd ..
cd ppg && pip install -r requirements.txt && cd ..
cd ups_hat && pip install -r requirements.txt && cd ..

Slave Crutch (Left) - Install Sensor Agents Only

cd tip_loadcell && make install && cd ..
cd ppg && pip install -r requirements.txt && cd ..
cd ups_hat && pip install -r requirements.txt && cd ..

Configure System

Copy Configuration File

Configure sensor calibration constants in mads.ini:

sudo cp templates/mads.ini /usr/local/etc/
sudo nano /usr/local/etc/mads.ini

Edit calibration values for your specific load cells. See Configuration Reference for details.

Configure Network

  1. Set static IP addresses on both boards (e.g., 10.42.0.1 for master, 10.42.0.2 for slave)
  2. Configure hostname for easy identification
  3. Ensure network connectivity between boards

Enable Systemd Services

Master Crutch (Right)

sudo cp templates/mads-broker.service /etc/systemd/system/
sudo cp templates/mads-web_server.service /etc/systemd/system/
sudo cp templates/mads-coordinator.service /etc/systemd/system/
sudo cp templates/mads-status_handler.service /etc/systemd/system/
sudo cp templates/mads-hdf5_writer.service /etc/systemd/system/
sudo cp templates/mads-eye_tracker.service /etc/systemd/system/
sudo cp templates/right/mads-tip_loadcell.service /etc/systemd/system/
sudo cp templates/right/mads-ppg.service /etc/systemd/system/
sudo cp templates/right/mads-ups.service /etc/systemd/system/

sudo systemctl daemon-reload
sudo systemctl enable mads-broker.service
sudo systemctl enable mads-web_server.service
sudo systemctl enable mads-coordinator.service
sudo systemctl enable mads-status_handler.service
sudo systemctl enable mads-hdf5_writer.service
sudo systemctl enable mads-tip_loadcell.service
sudo systemctl enable mads-eye_tracker.service
sudo systemctl enable mads-ppg.service
sudo systemctl enable mads-ups.service

Slave Crutch (Left)

sudo cp templates/left/mads-tip_loadcell.service /etc/systemd/system/
sudo cp templates/left/mads-ppg.service /etc/systemd/system/
sudo cp templates/mads-ups.service /etc/systemd/system/

sudo systemctl daemon-reload
sudo systemctl enable mads-tip_loadcell.service
sudo systemctl enable mads-ppg.service
sudo systemctl enable mads-ups.service

Verify Installation

After first boot, verify all services are running:

# Check service status (master)
sudo systemctl status mads-broker.service
sudo systemctl status mads-web_server.service
sudo systemctl status mads-coordinator.service

# View service logs
sudo journalctl -u mads-broker.service -f
sudo journalctl -u mads-web_server.service -f

# Test web interface
curl http://localhost:8000/api/status
Installation Complete! Your system is now ready. Access the web interface at http://master-ip:8000
