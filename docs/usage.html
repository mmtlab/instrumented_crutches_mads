Startup Sequence

Step 1: Power On Master Crutch (Right)

  1. Power on the right crutch
  2. Wait 30-45 seconds for all services to start
  3. Verify by checking system LED or accessing web interface

Step 2: Power On Slave Crutch (Left)

  1. Power on the left crutch
  2. Wait 20-30 seconds for sensor agents to start
  3. Check that left crutch connects to master (monitor network status)

Step 3: Connect Eye Tracker (Optional)

  1. Power on Pupil Neon glasses
  2. Ensure glasses connect to system WiFi (CrutchWiFi)
  3. Access web interface and verify eye-tracker status

Pre-Acquisition Calibration

Load Cell Calibration

Before each acquisition session, perform calibration to zero-offset the load cells:

  1. Access web interface at http://master-ip:8000
  2. Navigate to Record tab
  3. In Calibration section, click Calibrate button
  4. Hold both crutches level and elevated (no weight on ground)
  5. Wait for calibration to complete (2-3 seconds)
  6. Verify offset values update in interface
Note: Calibration should be done with crutches held steady. Repeat before each measurement session or if calibration drifts.

Web Interface Guide

Record Tab

Main interface for data acquisition:

  • Status: Current system state (Ready, Recording, Error)
  • Recording ID: Unique identifier for current session
  • Duration: Elapsed time since recording started
  • Condition: Current experimental condition
  • Start/Stop Buttons: Control recording
  • Condition Button: Change condition during recording

View Data Tab

Real-time visualization of sensor data:

  • Load cell forces over time
  • Eye gaze position
  • Heart rate and PPG waveforms
  • Live status of all agents

Download Tab

Export recorded data:

  • Browse available sessions
  • Download raw HDF5 files
  • Export to CSV format
  • Batch download multiple sessions

Recording Data

Starting a Recording Session

  1. Enter Subject ID and Session ID
  2. Verify all agents are healthy (green status indicators)
  3. Click Start button
  4. System will create unique recording ID and begin data acquisition
  5. Your timer will begin counting

During Recording

  • Monitor live data visualization to verify sensor functionality
  • If switching conditions, click Condition to mark transition
  • System automatically synchronizes all streams
  • Timestamps are microsecond-level synchronized across all sensors

Stopping a Recording Session

  1. Click Stop button
  2. System finalizes HDF5 file and indexes data
  3. Recording ID is stored for future reference
  4. Data is immediately available for download/export
Important: Do NOT power off crutches while recording. Always use Stop button to properly close data files.

Download & Export Data

Download HDF5 Format

Native format containing all raw data:

# File structure
subject_01_session_01_run_01.hdf5
├── metadata/
│   ├── subject_id
│   ├── session_id
│   └── timestamp
└── data/
    ├── tip_loadcell_left
    ├── tip_loadcell_right
    ├── handle_loadcell_left
    ├── handle_loadcell_right
    ├── ppg_left
    ├── ppg_right
    └── eye_tracker

Export to CSV

Tabular format for analysis in Excel/Python:

  1. Navigate to Download tab
  2. Select session(s)
  3. Choose "Export as CSV"
  4. Each sensor stream becomes separate CSV file

Shutdown Sequence

Normal Shutdown

  1. Complete all recordings (use Stop in web interface)
  2. Disconnect eye tracker (optional)
  3. Shutdown left crutch first: sudo poweroff
  4. Wait 5 seconds
  5. Shutdown right crutch: sudo poweroff
  6. Wait 30 seconds before power removal

Emergency Shutdown

If system becomes unresponsive:

  1. Force stop master web server: sudo systemctl stop mads-web_server.service
  2. Stop all agents: sudo systemctl stop 'mads-*.service'
  3. Power off: sudo poweroff
Pro Tip: Always review the Troubleshooting Guide if any issues occur during acquisition.
